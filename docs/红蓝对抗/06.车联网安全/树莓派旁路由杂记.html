树莓派旁路由杂记

00 背景

办公室的树莓派之前和警校合作搞了几个安全场景演示完就吃灰了，周末没事，研究了下树莓派+openwrt+openclash+AdGuard实现的内网旁路由。可玩性和稳定性也挺不错，简单写写给表哥们分享下。

01 效果

先上效果

  1. 连接wifi直接DHCP自动修改网关为旁路由，无感连接。

  1. 基于openclash实现gfw的ip自动分流。

  1. 简单的内网蜜罐监控配合server酱（其他IM机器人）实现内网ssh爆破告警和web登录告警。

  1. DNS 全局走的tls或者https提高隐私性，并且所有的dns记录都可追溯。

  1. 再就是一些统计性质的图标，可以满足一些监控的趣味。

02 教程

全部步骤均没有编译安装的环节（趟了一遍坑）对新手友好。

1. 树莓派安装官方系统。

下载raspberry PI官方烧录软件插入SD卡选择raspberry PI OS（64-BIT）直接一键化烧录操作系统。

2. 插入树莓派上电，启动，接入ssh，安装docker和docker版本的openwrt

2.1 运行 ifconfig 命令查看网卡信息，例如 网络接口逻辑名称是 enp3s0

开启此网卡混杂模式。

混杂模式就是接收所有经过网卡的数据包，包括不是发给本机的包。默认情况下网卡只把发给本机的包（包括广播包）传递给上层程序，其它的包一律丢弃。简单的讲,混杂模式就是指网卡能接受所有通过它的数据流，不管是什么格式，什么地址的。当网卡处于这种”混杂”方式时，该网卡具备”广播地址”，它对所有遇到的每一个数据帧都 产生一个硬件中断以便提醒操作系统处理流经该物理媒体上的每一个报文包。

ip link set eth0 promisc on

2.2 配置MACVLAN

docker network create -d macvlan --subnet=192.168.2.0/24 --gateway=192.168.2.1 -o parent=eth0 openwrt0

注: 192.168.2.0/24 和 192.168.2.1 自行调整。需要和你的路由器一个子网掩码。

2.3 查看docker网络

docker network ls

可以看见 openwrt0 网络。

2.4 创建网络配置文件

mkdir -p /home/docker/openwrt
vim /home/docker/openwrt/network

network 配置文件如下：

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config globals 'globals'
        option packet_steering '1'

config interface 'lan'
        option type 'bridge'
        option ifname 'eth0'
        option proto 'static'
        option netmask '255.255.255.0'
        option ip6assign '60'
        option ipaddr '192.168.2.100'
        option gateway '192.168.2.1'
        option dns '192.168.2.1'

config interface 'vpn0'
        option ifname 'tun0'
        option proto 'none'

修改 ipaddr，gateway，dns 为需要的地址。(dns后面会自动改)

2.5 拉取镜像并初始化

docker run -d --restart always --network openwrt0 --ip 192.168.2.100 --privileged --name openwrt sulinggg/openwrt:latest /sbin/init

注: 替换 192.168.2.100 为 openwrt 的 IP。

如果提示 WARNING: IPv4 forwarding is disabled. Networking will not work，需要进行开启 ipv4 转发。

vim /etc/sysctl.conf 
#添加 net.ipv4.ip_forward = 1
sysctl -p /etc/sysctl.conf

浏览器登录已经配置的 IP 并完成配置，账号密码 root/password。
